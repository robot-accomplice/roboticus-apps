# World Building

## Consistency Principles

- Once established, a fact about the world is permanent unless explicitly contradicted by an in-world event
- NPCs remember previous interactions with the player characters
- Distances, travel times, and geography remain consistent
- Political structures and power dynamics evolve based on player actions and world events
- Magic follows established rules — don't invent new rules mid-session

## Location Tracking

When introducing a new location, establish:
- Name and type (city, dungeon, wilderness, tavern)
- Key sensory details (what it looks, sounds, smells like)
- Notable inhabitants or dangers
- Relationship to other known locations (distance, direction, political ties)

Store each significant location in memory for future reference.

## Faction Tracking

For each faction, track:
- Name and alignment/goals
- Key figures (leaders, agents the party has met)
- Current disposition toward the party (friendly, neutral, hostile)
- Resources and influence
- Conflicts with other factions

## Improvisation Guidelines

When the player goes somewhere unexpected:
1. Use established world facts to extrapolate what would logically be there
2. Maintain the tone and danger level appropriate to the region
3. Create connections back to existing plot threads when possible
4. Store anything you improvise so it becomes permanent world fact

## Names and Culture

- Use consistent naming conventions within cultures/regions
- Different regions should feel culturally distinct (speech patterns, customs, architecture)
- Don't reuse NPC names unless deliberately referring to the same person
